# Week 2 – Log Analysis

## Overview

This repository contains the Week 2 SOC internship tasks completed as part of the Cyart internship program. The focus of this week was on alert prioritization, incident classification, alert triage, incident response workflows, and end-to-end incident documentation.

The work demonstrates SOC analyst thinking through documentation, structured workflows, and incident handling practices.

## Contents

- **Alert-Prioritization.md**

  Defines alert severity levels using CVSS, impact, urgency, and business context.

- **Incident-Classification.md**

  Classifies security incidents using MITRE ATT&CK techniques and contextual metadata.

- **Incident-Response.md**

  Documents the incident response lifecycle following NIST guidelines.

- **Alert-Triage.md**

  Demonstrates alert triage, prioritization, and threat intelligence validation.

- **Capstone-Incident-Report.pdf**

  Full alert-to-response incident report including executive summary, timeline, response actions, and recommendations.

- **Stakeholder-Briefing.md**

  Non-technical summary prepared for management and stakeholders.

## Tools & Frameworks Referenced

- SIEM concepts (Splunk-based SOC workflow)

- MITRE ATT&CK Framework

- NIST Incident Response Lifecycle

- CVSS Scoring System

- Threat Intelligence (AlienVault OTX – conceptual validation)

## Key Learnings

- Alert prioritization based on impact and exploitability

- Incident classification and MITRE mapping

- SOC alert triage and escalation decision-making

- Incident response workflow and documentation

- Communicating security incidents to non-technical stakeholders
